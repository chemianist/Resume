MOF Defect Screening Pipeline

RSC Advances, 2020

Metal-organic frameworks (MOFs) are promising for gas storage and separation, but structural defects can drastically alter properties. We developed a high-throughput Python pipeline to automatically detect misbonded atoms and coordination defects in thousands of MOF structures from the CoRE database.

Methods

  • Parsed CIF files and applied bonding criteria based on atomic radii and coordination chemistry
  • Identified miscoordinated metal nodes and missing linkers
  • Established a clean database of computational-ready MOFs with forcefield parameters (UFF, Dreiding)
  • Validated with DFT calculations on representative defective structures

Outcome

  • Over 2,500 MOFs with previously unreported defects were flagged, enabling more accurate property predictions
  • Open-source pipeline now used by several research groups for MOF screening
Publications:
T. Chen, T. A. Manz, "Identifying misbonded atoms in the 2019 CoRE metal-organic framework database", RSC Adv, 2020.
T. Chen, T. A. Manz, "A collection of forcefield precursors for metal-organic frameworks", RSC Adv, 2019.

Understanding Neural Networks

A simple, visual explanation of how machines learn from data

1. The Big Idea

A neural network is a system that learns patterns from examples. Instead of following fixed rules, it improves by making predictions, checking how wrong it was, and adjusting itself over time.

You can think of it like learning a skill: practice → feedback → improvement.

2. How Information Flows

Neural networks are made of layers:

  • Input layer: receives raw data (e.g., image pixels)
  • Hidden layers: process and transform the data step by step
  • Output layer: produces the final prediction

Each neuron performs a small calculation and passes its result forward. Many small decisions combine into one final answer.

3. Learning from Mistakes

At first, the model makes poor predictions. It then compares its prediction to the correct answer using a loss function, which measures the error.

The model uses this error to adjust its internal parameters through a process called backpropagation, gradually improving accuracy.

This process repeats many times:

Predict → Compare → Adjust → Repeat

4. Example: Recognizing a Cat

Imagine training a model to recognize cats in images.

  • Early layers detect simple features like edges and colors
  • Middle layers combine them into shapes and textures
  • Final layers recognize higher-level patterns like faces

Instead of being told what a cat looks like, the network learns from many examples.

5. Why Neural Networks Are Powerful

  • They can handle complex, high-dimensional data
  • They automatically learn useful features
  • They improve with more data and training

This makes them useful for tasks like image recognition, speech processing, and recommendation systems.

6. Key Takeaway

A neural network is not “programmed” with exact rules. Instead, it learns from experience, improves through feedback, and builds understanding step by step.

Even though the math can be complex, the core idea is simple:

Learn from examples, improve from mistakes, repeat.

Bias in AI for Computational Chemistry

A practical explainer on how human and data bias can shape scientific AI models

1. Why This Matters

AI models in computational chemistry are often treated as objective tools, but they inherit the assumptions built into their training data, feature choices, and evaluation methods. In chemistry, that matters because model outputs may influence which molecules are prioritized, which materials are screened, and which scientific conclusions seem most promising.

2. Where Bias Enters the Workflow

Bias can appear at multiple stages of a computational chemistry pipeline:

  • Dataset construction: training data may overrepresent certain molecule classes, chemistries, or property ranges.
  • Label generation: computed labels can reflect method limitations, approximation choices, or inconsistent reference standards.
  • Feature design: descriptors may emphasize some chemical behaviors while missing others.
  • Model evaluation: aggregate accuracy can hide poor performance on underrepresented chemical families.
  • Deployment decisions: a model may be used outside the domain where it was trained and validated.

3. Example: Molecular Property Prediction

Imagine training a model to predict solubility, binding affinity, or electronic properties. If the training set contains mostly small organic molecules with limited functional-group diversity, the model may appear accurate overall but perform poorly on more unusual compounds, charged systems, or larger molecules.

In that case, the issue is not only model quality, but also representation. The model has learned patterns from a narrow chemical space and may generalize poorly beyond it.

4. Human Bias in Scientific AI Decisions

Human bias does not disappear when AI is introduced. Scientists still decide which data to include, which methods to trust, which metrics to optimize, and which predictions deserve follow-up. Those choices can favor familiar chemistries, well-studied systems, or outcomes that align with existing expectations.

In computational chemistry, this can happen when researchers repeatedly benchmark against a narrow set of standard molecules, or when promising but less familiar regions of chemical space are excluded because they do not resemble historical training examples.

5. How to Reduce Bias

Reducing bias in computational chemistry AI requires both technical and leadership decisions:

  • Audit the dataset: examine which molecule classes, property ranges, and chemistries are over- or underrepresented.
  • Use subgroup evaluation: report performance across molecule families, charge states, or application domains rather than only overall error.
  • Document assumptions: make method limits, label uncertainty, and descriptor choices explicit.
  • Validate domain fit: confirm that the model is being used within the chemical space it actually learned.
  • Encourage interdisciplinary review: combine chemical intuition, data science, and domain expertise when interpreting model outputs.

6. Key Takeaway

AI does not remove human bias from computational chemistry. It can scale it, hide it, or amplify it if the data and evaluation process are not carefully designed. A strong chemistry AI workflow is not only accurate, but also transparent about what the model has seen, where it performs reliably, and where caution is needed.

In simple terms: better science requires both better models and better judgment.
